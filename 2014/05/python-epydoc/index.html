使用epydoc自动生成python项目文档

发表于 2014-05-25   |   分类于 python   |  

epydoc是一个能够根据python docstring自动生成api文档的工具，你只要在代码中按照规范写好注释，就能自动生成文档了，比较方便

1.安装

download，windows用户下载直接安装就可以；linux用户，提供了rpm安装包和源码安装两种方式，因为我是在windows系统上安装的，linux安装就不说了。

2.使用

###2.1可以使用命令行的方式启动

参数解释

  1. [–html|–pdf] 默认是生成html，如果指定pdf，生成的是.tex的文件并不是.pdf文件
  2. [-o DIR] 输出文件夹的路径
  3. [–parse-only|–introspect-only] 默认是使用两种方式获取docstring，你可以指定为某一种。parse是解析源码获得doc，introspect是使用内省来获得doc
  4. [-v|-q] 我没看明白
  5. [–name NAME] 项目文档名称
  6. [–url URL] 项目url链接
  7. [–docformat NAME] doc格式，默认是Epytext格式，此外还有javadoc，Plaintext和reStructuredText格式。我对javadoc比较熟悉，就使用了javadoc格式
  8. [–graph GRAPHTYPE] 提供3种可选，classtree显示基类和子类；callgraph,显示调用关系；umlclass以uml风格显示子类和基类
  9. [–inheritance STYLE]继承风格，grouped,listed,included,这个影响的是子类文档页面继承自父类的属性和方法展示风格不用，这个就随意了。
  10. [–config FILE] 你可以把配置信息写到一个文件里面，用这个参数指定参数文件

3.其他

中文乱码问题

这个工具，对中文支持可能不是太友好，我用这个工具生成文档的时候，我的代码所有模块都指定为了utf-8编码，但是，当我的package深度到第四级的时候，比如xxx.api.sale.NationQuery，NationQuery模块出现了中文乱码，前面3级模块都没问题，后来看epydoc源码找到docintrospecter.py这个文件，将line 567,源码中是ascii,改为utf-8，try: return unicode(docstring, 'utf-8')。问题解决，目前尚不清楚是bug还是其他原因
