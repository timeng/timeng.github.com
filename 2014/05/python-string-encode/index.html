Python2.x VS 3.x 字符编码处理

发表于 2014-05-29   |   分类于 python   |  

1. python2.x

在python2.7.6 环境测试

1.1 指定文件编码格式

在python2.x中，代码中任意位置出现中文，都应该在文件第一行或第二行指定编码方式，有两种方式，任选其一即可。否则会报错:SyntaxError: Non-ASCII character

# coding=utf-8
or
# -*- coding: utf-8 -*-

1.2 python2.x 字符串默认使用ascii编码

输出结果： ascii

1.3 非ascii编码，如中文字符，应统一使用unicode编码

不使用unicode，不能直接进行编码转换，比如

s2="中文test"
s2utf8=s2.encode("utf-8")

这样的代码，运行报错:UnicodeDecodeError: 'ascii' codec can't decode byte

使用unicode编码，有两种方式：

s2=u"中文test"
or
s2="中文test"
s2=unicode(s2,"utf-8")

以下，正确使用方法

输出结果：

{'a': u'\u4e2d\u6587test'}

中文test

2. python3.x

Python3.3.5 环境中测试

2.1 python3.x 默认文件格式是utf-8

python3.x中，文件前两行中的 编码格式设置，可以不用再写了，默认就是utf-8

2.2 默认字符串使用utf-8格式

输出结果: utf-8

2.3 python3.0 使用text类型替换了原来的unicode概念

python3.x中，你不能再使用u"中文"这样一个u前缀来表示unicode字符，所有的text类型，都是unicode，可以直接使用str.encode()进行编码，encode后变成 (binary) data类型，使用bytes.encode()解码为text

注:如果要表示一个binary date类型，你应该使用b""这样一个前缀

###
可以看到python3.x对于字符串、字符编码的处理，是非常友好的，当然，这种改变，也引起了python3不能向下兼容。但是，在可以预见的未来，会有越来越多的模块支持python3
